# Ralph Loop Progress Log

## Iteration 1 — Issue #30: Always enrich via Apify with Job Poster classification

### What was implemented

**`src/schemas.py`**
- Added `is_job_poster: bool = False` field to `Contact` schema.

**`src/core/scraper.py`**
- `normalize_brightdata_job` now sets `is_job_poster: True` on contacts created from a Bright Data `job_poster` field. The poster is stored at scrape time with its identity flag.

**`src/core/outreach.py`**
- Added `normalize_linkedin_url(url)` — canonicalizes any LinkedIn profile URL to `/in/{slug}`, stripping country subdomains, `www`, trailing slashes, and query params.
- Added `_poster_to_apify_item(poster)` — converts a Job Poster contact dict to an Apify-compatible item dict for batch LLM classification.
- Rewrote `source_contacts` to remove the early return that skipped Apify when existing contacts were present. New flow:
  1. Always calls Apify for a fresh Contact Sample.
  2. Extracts the Job Poster from existing contacts (`is_job_poster: True`).
  3. Deduplicates poster against Apify results by normalized LinkedIn URL; injects as synthetic extra when absent.
  4. Falls back to classifying the poster alone when Apify returns zero profiles.
  5. Sets `is_job_poster: True` on any classified contact whose URL matches the poster.
  6. Preserves `contacted: True` from previous contacts matched by normalized URL.

**`docs/adr/0005-always-enrich-via-apify-with-job-poster-classification.md`**
- New ADR documenting the decision to always run Apify on enrichment and include the Job Poster in the classification batch. Supersedes the "poster → skip Apify" shortcut from ADR 0002.

**Tests (13 new, 0 regressions — 122 total)**
- `test_outreach.py`: 7 tests for `normalize_linkedin_url` (subdomain, www, trailing slash, query params, empty, non-LinkedIn), 1 test verifying Apify is always called even with existing poster, and 5 tests covering poster injection, no-duplicate injection, Apify-failure fallback, `contacted` preservation, and `is_job_poster` flag propagation.
- `test_scraper.py`: 2 tests for `normalize_brightdata_job` — `is_job_poster: True` on poster contact, and no contacts when no poster.
