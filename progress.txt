# Ralph Loop Progress Log

## Iteration 1 — Contact Sample Cache on enrichment (#42)

Implemented the Contact Sample Cache as specified in ADR 0006 and issue #42.

**DB layer** (`src/db/cache.py`, `src/db/connection.py`):
- Added `contact_sample_cache` table (`company_slug` PK, `profiles` JSON, `fetched_at`, `display_name`) to `init_db`.
- New `src/db/cache.py` with `get_contact_sample`, `set_contact_sample`, `delete_contact_sample`.
- Added `log_activity(job_id, message)` public helper to `src/db/jobs.py`.
- Exported all new functions from `src/db/__init__.py`.

**Enrichment pipeline** (`src/core/outreach.py`, `src/pipelines.py`):
- `source_contacts` now checks the cache first (keyed by normalized company slug). On cache hit: skips Apify, logs company name + `fetched_at`, appends to the job's Job Activity Log. On cache miss: calls Apify, writes non-empty results to cache. Empty Apify results are never cached.
- Added `bust_cache=False` parameter to `source_contacts` (deletes cache entry before lookup when `True`).
- `run_enrichment_pipeline` accepts and forwards `bust_cache`.

**Tests** (`tests/test_contact_sample_cache.py`, `tests/test_outreach.py`):
- 16 new tests covering DB layer CRUD, cache-miss Apify call, cache-hit Apify skip, cache population, empty-result not cached, `classify_contacts` always runs, Task Logs log line, Job Activity Log append, `bust_cache` flow, slug normalization.
- Renamed `test_source_contacts_always_calls_apify_even_with_existing_job_poster` → `test_source_contacts_calls_apify_on_cache_miss_with_existing_job_poster`.
- All 238 tests pass.
