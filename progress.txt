# Ralph Loop Progress Log

## Iteration 1 — 2026-06-15

**Issue #58: Found and Accepted lanes with status migration**

### What was implemented

1. **Status enum** (`src/db/jobs.py`, `src/schemas.py`): Replaced `sourced`/`enriching`/`enriched`/`evaluating` with `found`/`accepted`. `VALID_STATUSES` now only accepts the new values.

2. **DB migration** (`src/db/connection.py`): `init_db()` runs two SQL migrations on upgrade:
   - `sourced` → `found`
   - `enriching`/`enriched` → `accepted`
   Also updated the default column value from `'sourced'` to `'found'`.

3. **Seed data** (`src/db/seed.py`): All seed jobs that previously had `status: "sourced"` now use `status: "found"`.

4. **Coordinator** (`src/core/enrichment/coordinator.py`): `ENRICHMENT_SOURCE_STATUSES` updated to `{"found", "accepted"}`. `begin_enrichment` moves a Found job to Accepted (no separate enriching lane); `abort_enrichment` is now a no-op (job stays Accepted).

5. **Enrichment DB operations** (`src/db/jobs.py`): `start_enrichment()` sets `accepted`; `enrich_job()` sets `accepted` (was `enriched`). Activity log no longer logs "Enrichment started" — uses generic "Moved X → Y".

6. **Scraper** (`src/core/scraper.py`): New jobs are created with `status: "found"`.

7. **Pipeline** (`src/pipelines.py`): Guard changed from `status != "enriching"` to `status != "accepted"`.

8. **Service** (`src/service/job_hunter.py`): `promote_sourced_jobs` now selects `status == "found"` jobs.

9. **API** (`src/web/server.py`): `PUT /api/jobs/{id}/status` returns 422 for obsolete statuses.

10. **Kanban Dashboard** (`src/web/dashboard.html`, `boardRenderer.js`, `drawerController.js`):
    - Replaced Sourced/Enriching/Enriched lanes with Found/Accepted.
    - Updated stages arrays in `initKanbanDnd` and `initLaneCollapse`.
    - Removed `newStatus === 'enriching'` routing hook from `moveJobStage` — lane drag is now status-only.
    - Removed archived-card-to-enriching guard (no enriching lane).
    - Updated status CSS classes, polling function, and drawer button conditions.

11. **Tests**: Added `tests/test_status_migration.py` (16 new tests); updated all existing tests to use new status values. All 409 tests pass.

## Iteration 2 — 2026-06-15

**Issue #59: Enrichment runs in place on Accepted**

### What was implemented

The core enrichment-in-place behavior (coordinator, pipeline, API) was already complete from iteration 1. This iteration added the one missing acceptance criterion: the on-card spinner badge.

1. **`cardEnrichingBadge(jobId, enrichingJobId)`** (`src/web/static/js/boardRenderer.js`): New exported pure function. Returns a spinner badge HTML string when `jobId === enrichingJobId` (and enrichingJobId is not null), otherwise returns `''`.

2. **Card template** (`src/web/static/js/boardRenderer.js`): The spinner badge is rendered inline in the company badge row of each Accepted card. `renderBoard` now reads `filters.enrichingJobId` and passes it to `cardEnrichingBadge` for each card.

3. **`activeEnrichJobId` tracking** (`src/web/dashboard.html`):
   - Added `let activeEnrichJobId = null` module-level variable.
   - `renderActiveVariant()` now spreads `{ enrichingJobId: activeEnrichJobId }` into the filters passed to `renderBoard`.
   - `enrichJob()` sets `activeEnrichJobId = data.job_id` immediately after the POST response, triggering a re-render with the spinner visible.
   - `refreshContacts()` does the same.
   - Both `onDone` and `onError` callbacks in both functions clear `activeEnrichJobId = null` and re-render, removing the spinner when the task finishes.

4. **Test** (`tests/test_dashboard_modules.py`): Added `test_board_renderer_shows_enriching_badge_for_active_task` — verifies `cardEnrichingBadge` returns a spinner for the matching job ID, empty string for a non-matching ID, and empty string when `enrichingJobId` is null.

All 410 tests pass.

## Iteration 4 — 2026-06-15

**Issue #61: Safe Apify fetch and append-only Contact Sample Cache**

### What was implemented

1. **`ApifyInfrastructureError`** (`src/core/enrichment/contact_sample.py`): New exception class for credential and API failures (no token, HTTP trigger error, no run ID, run FAILED/ABORTED/TIMED-OUT, dataset fetch error). Infrastructure failures propagate as exceptions; only a successful Apify run that returns 0 profiles returns `[]`.

2. **`_run_apify_actor` simplified** (`src/core/enrichment/contact_sample.py`): Removed the `company_slug_candidates` loop entirely. Now takes `company_url: str` as the only required argument and delegates directly to `_run_apify_for_company_page`. Raises `ApifyInfrastructureError` if no URL provided.

3. **Empty-result caching** (`src/db/cache.py`): Removed `if not profiles: return` guard from `set_contact_sample`. Empty Apify responses (zero employees from a successful run) are now cached to prevent repeat Apify calls.

4. **`source_contacts` updated** (`src/core/enrichment/source.py`):
   - Removed `bust_cache` parameter.
   - Added `elif not company_url:` branch: skips Apify, logs a warning, sets `meta["empty_reason"] = "no_company_url"`, and falls through to job poster classification.
   - Always calls `set_contact_sample` after a successful Apify run (including zero-profile results).
   - `ApifyTimeoutError` and `ApifyInfrastructureError` propagate up to the pipeline instead of being silently swallowed.

5. **Pipeline** (`src/pipelines.py`): Removed `bust_cache` parameter. Added `no_company_url` branch in the enrichment note logic that sets `"No LinkedIn company URL — cannot fetch employees."` when `source_meta` contains `empty_reason = "no_company_url"`.

6. **Service** (`src/service/job_hunter.py`): Removed `bust_cache` parameter from `complete_enrichment`.

7. **API server** (`src/web/server.py`): Removed `run_refresh_contacts_task_with_logs`, the `bust_cache` wrapper logic, and the `POST /api/jobs/{job_id}/refresh-contacts` endpoint. `run_enrichment_task_with_logs` simplified (no bust_cache).

8. **Dashboard** (`src/web/dashboard.html`): Removed `refreshContacts` JS function and its export from the window object.

9. **Drawer** (`src/web/static/js/drawerController.js`): Removed the Refresh Contacts button from the Accepted job drawer template.

10. **Tests**: Updated 14 tests across `test_contact_sample_cache.py`, `test_enrichment_modules.py`, `test_outreach.py`, `test_enrichment_pipeline.py` to reflect new behavior. Replaced `test_refresh_contacts.py` with removal-verification tests. Added 16 new tests covering: empty-list caching, missing companyUrl skips Apify, infrastructure errors not cached, `no_company_url` enrichment note, and removal of the refresh-contacts endpoint and JS. All 407 tests pass.
