# Ralph Loop Progress Log

## Iteration 1 — Issue #52: Manual archive on Rejected cards

### Summary
Implemented the first tracer bullet for Archived Jobs feature (Parent PRD #51).

**DB schema** (`src/db/connection.py`):
- Added `archived INTEGER DEFAULT 0`, `rejectedAt TEXT DEFAULT ''`, `autoArchiveExempt INTEGER DEFAULT 0` columns via migration-safe `ALTER TABLE` blocks.
- Backfill: after seed insert, sets `rejectedAt = datetime('now')` for any existing Rejected jobs with no timestamp.

**DB logic** (`src/db/jobs.py`):
- `_parse_job_row`: exposes `archived`, `rejectedAt`, `autoArchiveExempt` on every job dict.
- `get_jobs`: filters `archived = 0` by default — archived jobs are invisible on the active board.
- `update_job_status`: sets `rejectedAt` on first move to Rejected via SQL `CASE` (never overwrites existing value).
- `archive_job(job_id)`: new function — archives a Rejected job (`archived=1`), appends "Archived" to Job Activity Log; returns `None` for non-Rejected or nonexistent jobs.
- `job_exists`: unchanged — still matches archived rows for deduplication.

**Schema** (`src/schemas.py`): added `archived`, `rejectedAt`, `autoArchiveExempt` fields to `Job` Pydantic model.

**API** (`src/web/server.py`): new `POST /api/jobs/{id}/archive` endpoint — 200 on success, 422 for non-Rejected, 404 for missing.

**Dashboard** (`src/web/dashboard.html`):
- `getKanbanCardMovementButtons`: Rejected cards now show a purple box-archive icon on hover instead of the ban/reject button.
- `archiveJob(id)` async JS function: calls `POST /api/jobs/{id}/archive`, removes the card from `masterJobs`, and re-renders the board.
- CSS: `.kanban-action-btn.archive-btn:hover` — purple hover color.

**Tests** (`tests/test_archive.py`): 16 new tests covering DB schema, backfill, `rejectedAt` logic, `get_jobs` filtering, `archive_job` behavior, deduplication, and all three API cases. All 341 tests pass.
