# Ralph Loop Progress — PRD #86

## PRD #86: Gemini Batch API Evaluation & Scraped/Matched Lanes
- **Issue #87**: Implemented Scraped & Matched lanes + status migration.
    - Updated database schema and migration logic to map `found` to `scraped` (unscored) or `matched` (scored).
    - Updated frontend Kanban board with new lanes and styles.
    - Updated backend pipelines, schemas, and service logic to use the new status model.
    - Updated all relevant tests to align with the new status model.
- **Issue #88**: Migrated Gemini client to google.genai SDK.
    - Replaced `google-generativeai` with `google-genai` in requirements.txt.
    - Rewrote `gemini_client.py` and `gemini_agent.py` to use `google.genai.Client`.
    - Kept synchronous `generate_text` for reassess and outreach pipelines.
    - Added `generate_text_from_pdf` multimodal path for future Resume Import (#99).
    - Added `tests/test_gemini_client.py`; fixed pre-existing test regressions from #87 status migration.
- **Issue #89**: Submit-and-return search via chunked Batch Evaluation Jobs.
    - Search pipeline saves all new jobs to Scraped (`matchType` empty) before any LLM work, then returns immediately.
    - Added `submit_batch_evaluation` seam with JSONL keyed by `job_id`, `response_mime_type: application/json`, and 100-job chunking.
    - Added `batch_jobs` table + CRUD (`src/db/batch_jobs.py`); skips jobs already covered by in-flight batches.
    - Removed packed-prompt batch evaluation from `matcher.py` (backfill still uses sequential `evaluate_job` until #93).
    - Added `tests/test_batch_jobs.py`, `tests/test_batch_evaluation.py`; updated `tests/test_search_pipeline.py`.
    - pytest: 693 passed.
- **Issue #90**: Batch Poller write-back (Scraped → Matched/Rejected) + resume + cadence.
    - Added `src/core/batch_poller.py` with `poll_cadence_seconds`, `is_due_for_poll`, `collect_batch_results`, and `write_back_job_evaluation`.
    - On `JOB_STATE_SUCCEEDED`, parses JSONL results keyed by `job_id`, writes scores via `update_job_evaluation`, moves passing jobs to **Matched** and gate failures to **Rejected**.
    - Dashboard starts a background poller on startup (resumes in-flight `batch_jobs`; never auto-submits); `/api/batch-poller/logs` SSE for event-only Task Logs.
    - Persisted search preferences on `batch_jobs` (`searchRemoteTypes`, `searchSeniorities`) for attribute gating at write-back.
    - Added `tests/test_batch_poller.py` (cadence table, fake-client write-back, terminal failure).
    - pytest: 708 passed.
- **Issue #91**: Evaluation Lock + Cancel (dashboard + CLI).
    - Added `src/core/evaluation_lock.py` with derived lock status from non-terminal `batch_jobs` rows, `EvaluationLockError`, and `cancel_in_flight_batches()` (calls `batches.cancel()`, marks rows `JOB_STATE_CANCELLED`, jobs stay in Scraped).
    - Dashboard: `GET /api/evaluation-lock`, `POST /api/evaluation-lock/cancel`; search/scrape return 409 when lock active; "Assessing N jobs…" indicator with Cancel control; scrape button disabled while locked.
    - CLI: `--search` and `--backfill` refuse with clear message when lock active.
    - Added `tests/test_evaluation_lock.py` (lock query, cancel, API 409, CLI refusal, dashboard HTML scan).
    - pytest: 717 passed.
- **Issue #92**: Poison-job handling (batchAttempts → Unclassified fallback).
    - Added `jobs.batchAttempts` column (INTEGER, default 0) and `increment_batch_attempts()` DB helper.
    - Batch Poller increments attempts on malformed/failed per-request lines and whole-batch terminal failures; jobs stay in Scraped until 3 attempts.
    - After 3 failures: `apply_unclassified_fallback()` uses scraper `remoteType`/`seniority`, marks job Unclassified, moves to Matched (or Rejected on gate fail).
    - Unclassified badge already present on Kanban card and drawer (hover tooltip).
    - Added poison-job tests in `tests/test_batch_poller.py` (retry counting, fallback, whole-batch failure).
    - pytest: 721 passed.
- **Issue #93**: Backfill convergence + in-flight cap drip-feed.
    - Rewrote `run_backfill_pipeline` to use `submit_backfill_batches` (shared batch path, no inline `evaluate_jobs_batch`).
    - Added `MAX_IN_FLIGHT_BATCHES=15` cap; without `--wait` submits up to 15 chunks then returns; with `--wait` polls and drip-feeds remaining chunks as slots free.
    - CLI: `--backfill --wait` blocks until all in-flight batches reach terminal state.
    - Rewrote `tests/test_backfill_pipeline.py` for chunk counting, in-flight cap, and wait drip-feed.
    - pytest: 720 passed.
- **Issue #94**: --collect / --collect --wait CLI for headless collection.
    - Added `run_batch_collection`, `collect_in_flight_batches_once`, and `wait_for_in_flight_collection` in `src/core/batch_poller.py`.
    - `--collect` one-shot polls all in-flight batches (ignores cadence); `--collect --wait` loops on poll cadence until terminal.
    - Neither variant submits batches; both allowed while Evaluation Lock is active.
    - CLI: `run_collect` + `--collect` / `--wait` flags in `src/cli/cli.py`; service seam `collect_batch_evaluation_results`.
    - Added `tests/test_collect_cli.py` (write-back, lock-active, wait loop, no-op).
    - pytest: 724 passed.
